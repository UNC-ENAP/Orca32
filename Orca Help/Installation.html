INSTALLING ORCA

If you have the ORCA binary you can just drag it to the applications folder.

If you wish to build from source, you must first install (at a minimum) the National Instruments driver. Otherwise you will not be able to link because a number of header files will be missing.

Before you can run ORCA using any of the PCI objects you need to install one of the PCI drivers, either the 617 or the 620 driver depending on which type of PCI card you have. If the driver is not installed ORCA will crash as soon as a PCI card is used in the main configuration. Also you need to install the National Instruments driver to run any of the GPIB objects.

Driver installation packages can be found in the Drivers folder of the distribution.

Starting with CVS:

-copying the whole project from another source is OK unless you want to use cvs. Then you must get the Orca folder from the cvs server using cvs.

put the following in your .tcshrc file:

  • setenv CVSROOT :pserver:username@128.95.100.177:/Users/orca/cvs
  • setenv CVS_RSH ssh
  • setenv CVSEDITOR pico

Then log out and log back in or source the .tcshr file. Next copy the cvswrappers file into your home dir and rename it to .cvswrappers -- if you don't copy and rename the cvswrappers file, cvs will not be able to work with the .nib files or the xcode project. You can find the cvswrappers file on the distribution disk in the CVS Notes folder. Now cvs should be ready to work. The first time you use the Orca cvs server you must login:

  • cvs login

and enter your cvs password.

You shouldn't have to do a login again.

Note: We do not yet have anonymous cvs at this time. You must have an account on the server.

Getting Orca the first time
  • cd ~/Dev
  • rm -rf Orca (if you copied the Orca folder from the distribution CD, you must remove it before you can checkout the source)
  • cvs checkout Orca

double-click on the Orca.xcode file to launch project builder and compile the code.

You should be ready to go.

Note that the Drivers folder and the OrcaPalette folder are not under cvs at this time. You can copy them from the distribution CD.

Updating

To do a normal update with no new files you can use the project builder cvs functions. To grab new files,or to grab a new version of the project it's a little tricky. do the following

  • cd ~/Dev/Orca
  • rename your Orca.xcode file to something else or delete it (rm -rf Orca.xcode)
  • cd ..
  • cvs update -d Orca

Following the above steps will ensure that all new folders and files will be created and a new xcode project will be retrieved which will have all the right files included.

Getting a specific version

Starting with version 2.6.6 we started tagging the versions in cvs. To retrieve a specific version do the following:

  • cd ~/Dev/Orca
  • rename your Orca.xcode file to something else or delete it (rm -rf Orca.xcode)
  • cd ..
  • cvs update -r v2-6-6 Orca

Of course, the 'v2-6-6' should be replaced with the desired version, i.e. for ORCA verion 2.6.7 you would use v2-6-7 and so on. Once you have updated to a specific version, cvs will not allow you to commit changes. To return to the current version, you must remove the version tag and do a full update:

  • cd ~/Dev
  • cvs -q update -A Orca
  • cd ~/Dev/Orca
  • rename your Orca.xcode file to something else or delete it (rm -rf Orca.xcode)
  • cd ..
  • cvs update -d Orca

Fixing the sticky tag

If you revert back to a specific version of a file, cvs views that revision as a snapshot of history and will not allow you to commit changes to it. Before you make changes you need to clear the sticky tag:

  • cd to where the file is
  • cvs -l update -A fileName
  • cvs update -d fileName

Then the file can be modified and commited.

Orca Palette

The Orca Palette project is an Interface Builder palette. The resulting palette can be added to Interface Builder (use the Prefences/Palettes dialog to add) to build new dialogs with the Orca widgets.
